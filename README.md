# АИС «FREEDOM»: учёт абонементов танцевальной студии


**Версия:** 1.0  
**Статус:** в эксплуатации  
**Целевая аудитория:** администраторы и менеджеры танцевальной студии FREEDOM

## О системе

АИС (автоматизированная информационная система) «FREEDOM» — это веб‑приложение для учёта абонементов клиентов танцевальной студии. Система автоматизирует ключевые процессы: продажу и продление абонементов, контроль посещений, аналитику и отчётность.

**Цель системы:**  
- упростить и ускорить работу администраторов;  
- исключить ошибки ручного учёта;  
- обеспечить прозрачность финансовых операций;  
- повысить качество обслуживания клиентов за счёт оперативного доступа к данным.

## Ключевые возможности

Ниже — детальный разбор **реализованных** функций системы.

### 1. Продажа и оформление абонементов

- **Создание нового абонемента:**  
  - выбор услуги (абонемента) из выпадающего списка с поиском;  
  - автоматическое заполнение даты продажи (текущая дата);  
  - расчёт даты окончания на основе длительности услуги (дни);  
  - отображение информации об услуге (название, цена, длительность) в реальном времени.  
- **Валидация данных:**  
  - проверка обязательных полей (например, «Максимальное количество посещений»);  
  - блокировка отправки формы при ошибках;  
  - визуальная индикация ошибок (подсветка полей, модальные окна).
- **Уведомления:** всплывающие тосты (`toast`) об успехе или ошибке операции.

### 2. Управление абонементами (редактирование)

- редактирование существующих записей об абонементах;  
- синхронизация видимого и скрытого полей даты окончания;  
- обновление данных об услуге при смене типа абонемента (через AJAX‑запрос).

### 3. Контроль посещений

- отметка посещений клиентов прямо на странице системы;  
- автоматическое обновление счётчика оставшихся посещений;  
- визуальное изменение состояния кнопок «Отметить посещение» в зависимости от лимита посещений;  
- просмотр истории посещений клиента в модальном окне (без перезагрузки страницы).

### 4. Поиск и фильтрация данных

- поиск продаж абонементов по ключевым полям (клиент, дата и т. д.) через AJAX:  
  - мгновенное обновление таблицы без перезагрузки;  
  - подсветка найденных результатов;  
  - сообщение «нет результатов», если ничего не найдено.  
- аналогичные механизмы поиска для клиентов и сотрудников.

### 5. Финансовая аналитика и отчётность

- отображение статистики по продажам и доходам;  
- фильтрация данных по датам (с преобразованием в UTC для запросов);  
- обновление блока статистики через AJAX при смене фильтров;  
- модальное окно для подтверждения удаления финансовых транзакций (с заполнением полей данными).

### 6. Управление пользователями и аутентификация

- форма входа и регистрации с единым интерфейсом;  
- переключение между вкладками «Вход» / «Регистрация»;  
- валидация полей при регистрации (обязательные поля, совпадение паролей);  
- визуальная обратная связь при ошибках (анимация «тряски»).

### 7. Оповещения и мониторинг

- автоматические оповещения об абонементах с истекающим сроком действия;  
- карточки‑оповещения с цветовой индикацией статуса:  
  - **нормальный** (зелёная граница);  
  - **предупреждение** (жёлтая граница);  
  - **критический** (красная граница, фон).  
- круговые индикаторы прогресса для отображения процента использования абонемента;  
- возможность быстрого продления абонемента из карточки оповещения.

### 8. Административные функции

- пагинация списков (клиенты, абонементы, транзакции) с адаптивными кнопками;  
- загрузка и отображение данных через AJAX (без перезагрузки страницы);  
- подтверждение удаления записей (клиентов, транзакций) через модальные окна.

### 9. Интерфейс и UX

- адаптивный дизайн для ПК и мобильных устройств;  
- плавные анимации появления элементов (уведомления, карточки, модальные окна);  
- эффекты при наведении (подъём кнопок, карточек, изменение теней);  
- фиксированный сайдбар с навигацией (открытие/скрытие по кнопке);  
- экран‑загрузчик при инициализации системы;  
- поддержка доступности (`prefers-reduced-motion` — отключение анимаций).

## Технические особенности

- **Интерфейс:** одностраничное веб‑приложение (SPA‑подобное поведение за счёт AJAX).  
- **Технологии:** HTML, CSS (модульная структура), JavaScript, Bootstrap.  
- **Взаимодействие с сервером:** AJAX‑запросы для загрузки/обновления данных.  
- **Стилизация:** модульная CSS‑архитектура (разделение на `base/`, `components/`, `layout/`, `responsive/`).  
- **Навигация:** сайдбар, верхняя панель, кликабельные карточки.

## Как начать работу


1. Откройте браузер и перейдите по адресу системы.  
2. Авторизуйтесь с учётными данными администратора.  
3. Используйте навигацию (сайдбар/верхняя панель) для доступа к нужным разделам:  
   - «Продажи абонементов» → создание/редактирование;  
   - «Посещения» → отметка посещений, просмотр истории;  
   - «Финансы» → аналитика, транзакции;  
   - «Клиенты» → поиск, управление данными.
4. Пользуйтесь поиском и фильтрами для быстрого нахождения информации.
5. Обращайте внимание на уведомления (тосты) и цветовые индикаторы статусов.
